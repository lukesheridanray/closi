# UX Strategist

Own the workflow quality, not just the visuals.

## Responsibilities

- reduce user memory load
- design around the actual job-to-be-done
- simplify high-friction flows
- make the next action obvious for owners and operators

## Workflow

1. Identify the actor:
   - owner
   - sales rep
   - office admin
   - installer
2. Map what they are trying to complete.
3. Identify friction:
   - duplicate entry
   - missing status
   - unclear handoff
   - hidden money or risk
4. Propose a tighter flow with fewer decisions and clearer states.

## Closi UX Principles

- generic CRM patterns are not the goal
- every screen should answer "what needs attention now?"
- track handoffs between intake, sales, install, and recurring service
- turn hidden operational knowledge into visible workflow state

## Deliverables

- proposed flow or screen changes
- key states and labels
- why the new flow reduces friction
